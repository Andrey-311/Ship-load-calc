## Описание проекта

**Ship Load Calculator** — desktop-приложение для расчёта нагрузки масс судов в соответствии с нормативным документом **ОСТ5Р.0206-2002** «Нагрузка масс гражданских и вспомогательных судов. Классификация элементов нагрузки».

Приложение предназначено для инженеров-конструкторов и расчётчиков, участвующих в проектировании судов, доков, плавучих сооружений и земснарядов.

### Основные возможности

- **Параллельная работа** — несколько конструкторов могут одновременно вводить данные по разным элементам судна
- **Заявочная система** — изменения группируются в заявки (ECR) и проходят подтверждение расчётчиком
- **Автоматическая агрегация** — система сама суммирует массы по иерархии кодов (раздел → группа → подгруппа → статья → подстатья)
- **Вариантность запасов** — расчёт дедвейта для 100%, 50%, 10% и пользовательских вариантов
- **Экспорт в Excel** — формирование сводной таблицы по образцу из нормативной документации

### Роли пользователей

| Роль | Обязанности |
|------|-------------|
| **Конструктор** | Создаёт заявки, вводит массы и координаты элементов, отправляет на подтверждение |
| **Расчётчик** | Подтверждает или отклоняет заявки, выполняет сводные расчёты, экспортирует отчёты |
| **Администратор** | Управляет проектами, загружает справочник кодов, настраивает параметры |

## Функциональные возможности

### Клиент 1 — Конструктор (ShipDesigner.exe)

- Создание и редактирование заявок на изменение нагрузки
- Табличный ввод строк с кодом, массой, координатами X, Y, Z
- Отображение суммарного центра тяжести текущей заявки
- Просмотр активной нагрузки (все подтверждённые заявки)
- Отправка заявки на подтверждение

### Клиент 2 — Расчётчик (ShipCalculator.exe)

- Просмотр списка заявок на подтверждение
- Подтверждение/отклонение заявок с проверкой отрицательной массы
- Иерархическое дерево активной нагрузки (с группировкой по кодам)
- Сводная таблица «Нагрузка масс» в формате ОСТ
- Расчёт водоизмещения порожнем (Lightweight) и дедвейта (Deadweight)
- Применение типовых случаев нагрузки (100%, 50%, 10% запасов)
- Создание и редактирование типовых случаев
- Экспорт сводной таблицы в Microsoft Excel (.xlsx)

### Серверная часть (FastAPI)

- REST API для работы с проектами, заявками, строками нагрузки
- Агрегация масс по иерархии кодов
- Автоматический расчёт Lightweight (разделы 01–13 согласно п. 3.7 ОСТ)
- Расчёт Deadweight с учётом процентов по разделам 08, 14–18
- Расчёт объёмного водоизмещения (масса / плотность воды)
- Валидация координат по габаритам судна
- Проверка отрицательного Lightweight при подтверждении заявок

## Технологический стек

| Компонент | Технология | Причина выбора |
|-----------|------------|----------------|
| **Backend** | Python 3.12 + FastAPI | Асинхронность, автодокументация, высокая производительность |
| **База данных** | SQLite (разработка) / PostgreSQL (production) | Простота на старте, лёгкий переход на серверную БД |
| **ORM** | SQLAlchemy 2.0 | Асинхронная поддержка, абстракция от БД |
| **Миграции** | Alembic | Поддержка SQLite → PostgreSQL |
| **GUI** | PySide6 (Qt for Python) | Профессиональные виджеты, LGPL лицензия |
| **HTTP клиент** | httpx | Асинхронная поддержка, интерфейс как у requests |
| **Экспорт Excel** | openpyxl | Работа с .xlsx, форматирование ячеек |
| **Тестирование** | pytest + pytest-asyncio + pytest-qt | Единый стандарт для всех компонентов |
| **Сборка .exe** | PyInstaller | Кроссплатформенная упаковка |
| **Контроль версий** | Git (GitFlow) | Стандарт индустрии |
| **CI/CD** | GitHub Actions | Автоматические тесты и сборка |

## Установка и запуск

### 1. Установка Python

1. Перейдите на [python.org/downloads](https://python.org/downloads)
2. Скачайте **Python 3.12** для Windows (64-bit)
3. При установке **обязательно** отметьте опцию:
   - ✅ **Add Python to PATH**

Проверьте установку:
```cmd
python --version
2. Клонирование репозитория
cmd
git clone https://github.com/your-org/ship-load-calculator.git
cd ship-load-calculator
3. Создание виртуального окружения
cmd
python -m venv venv
venv\Scripts\activate
После активации в начале строки появится (venv).

4. Установка зависимостей
Используйте управляющий скрипт:

cmd
run.bat install
Или вручную:

cmd
cd backend
pip install -r requirements-dev.txt
cd ..\clients\client1
pip install -r requirements.txt
cd ..\client2
pip install -r requirements.txt
cd ..\..
pre-commit install
5. Настройка базы данных
Создайте файл .env в корне проекта:

env
DATABASE_URL=sqlite+aiosqlite:///./data/ship_load.db
ENVIRONMENT=development
API_PORT=8000
AUTH_ENABLED=false
Примените миграции:

cmd
cd backend
alembic upgrade head
cd ..
6. Загрузка справочника кодов
Справочник кодов загружается через административный интерфейс (Клиент 2) в формате CSV или JSON.

Пример CSV:

csv
code,name,level,parent_code
01,Корпус,1,
0101,Корпус металлический,2,01
010101,Обшивка наружная настил второго дна,3,0101
01010101,Обшивка наружная,4,010101
0101010101,Обшивка днища с набором,5,01010101
7. Запуск приложения
Запуск серверной части
В отдельном терминале:

cmd
run.bat backend
Сервер запустится на http://localhost:8000

Запуск Клиента 1 (Конструктор)
В другом терминале:

cmd
run.bat client1
Запуск Клиента 2 (Расчётчик)
В третьем терминале:

cmd
run.bat client2

Тестирование
Запуск всех тестов
cmd
run.bat test
Запуск отдельных групп
cmd
cd backend
pytest tests/unit                    # Только модульные тесты
pytest tests/integration             # Только интеграционные тесты
pytest tests/unit/test_aggregation.py -v  # Конкретный файл
Покрытие кода тестами
cmd
cd backend
pytest --cov=app --cov-report=html
Отчёт будет в backend/htmlcov/index.html

cmd

cmd
